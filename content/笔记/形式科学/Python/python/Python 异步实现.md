##### Python 异步实现
- **多进程异步**
	- 并行执行：多进程可以利用多个 CPU 核心并行执行任务，每个进程都有自己独立的内存空间，相互之间不受影响。
	- 资源消耗：每个进程都有自己独立的内存空间，因此在一定程度上会消耗更多的系统资源，如内存。
	- 进程间通信：多进程之间通信相对复杂，需要使用诸如队列、管道等机制来实现进程间的数据传递和同步。
- **多线程异步**
	- 并发执行：多线程可以实现并发执行任务，每个线程都有自己的执行流程，可以同时执行多个任务。
	- GIL 限制：在 CPython 解释器中，全局解释器锁（GIL）限制了同一时刻只能有一个线程执行 Python 字节码，因此多线程并不能有效地利用多核 CPU。
	- 共享内存：多线程共享进程的内存空间，因此在共享数据时需要注意线程安全问题，例如使用锁机制来保护共享资源。
- **协程异步**
	- 单线程：协程是在单线程中实现的，通过事件循环调度协程的执行，因此不需要多个线程来实现并发。
	- 非阻塞 I/O：协程适用于 I/O 密集型任务，可以通过非阻塞 I/O 来实现并发执行，避免了线程切换的开销和 GIL 的限制。
	- 轻量级：协程比多进程和多线程更轻量级，创建和切换协程的开销比创建和切换线程的开销要小得多。
##### 进程线程协程
- **进程线程协程**
	- 线程是操作系统能够进行运算调度的最小单位。大部分情况下，它被包含在进程之中，是进程中的实际运作单位。一条线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务
	- 进程是计算机中的程序关于某数据集合上的一次运行活动，是系统进行资源分配和调度的基本单位
	- 协程是一种特殊的函数，可以在执行过程中被挂起并暂停，然后在需要的时候恢复执行。协程的使用能够实现异步编程，避免了阻塞，提高了程序的并发性能。
- **进程线程运行状态**
	- 阻塞是指线程或进程由于某种原因而无法继续执行，需要等待某些条件满足后才能继续执行的状态。
	- 非阻塞是指线程进程在执行过程中，不会因为某个操作的未完成而被暂停，而是会继续执行其他操作，不等待当前操作完成。在非阻塞模式下，任务会周期性地检查某个操作是否已就绪，一旦就绪就会继续执行，而不会阻塞在那里等待操作完成。
- **进程线程任务状态**
	- 并发把任务在不同的时间点交给处理器进行处理。在同一时间点，任务并不会同时运行
	- 并行把每一个任务分配给每一个处理器独立完成。在同一时间点，任务一定是同时运行
